# 星詠み 自動運用の仕組み

運用ゴール: **収益化ファネルを自動運用し、手動が必要な箇所は毎朝「あと1クリック」の状態まで用意する。**

## 毎朝9時に自動で起きること(~/side_hustle_monitor/monitor.py)

1. **死活監視** — 全5ページ(トップ/星座/相性/運命数/タロット)の配信とコンテンツを確認
2. **収益化ステップの自動検出** — 本番サイトをスキャンして進捗を判定し、「👉 次の一手」を1つだけレポートに表示
   - アフィリエイト実装済みか(本文中のrakuten/amzn/a8.netリンクを検出)
   - 独自ドメイン移行済みか(canonicalのgithub.io有無を検出)
   - AdSense設置済みか(adsbygoogleを検出)
   - Xアカウント作成済みか(自動検出不可のためフラグファイル参照)
3. **本日のX投稿文を自動生成** — `data/hoshiyomi_today_post.txt` に保存し、レポートに
   **ワンクリックで投稿画面が開くURL** を記載(残る手作業は「投稿」ボタンだけ)

## 投稿文のローテーション(日付シードで決定論的)

| 曜日 | 内容 |
|---|---|
| 月〜金 | 12星座ランキングTOP3(seiza.htmlの表示と完全一致することを検証済み) |
| 土 | 相性占いの訴求 |
| 日 | 今日の一枚(タロット)+ラッキーカラー |

ランキングはJS(seiza.html)とPython(monitor.py)で同一の32bitハッシュ
(murmur3 finalizer)を実装しており、`node`/`python3`/実ブラウザの3系で
出力一致を確認済み(2026-07-05)。**片方を変更したら必ず両方を変更すること。**

## 手動ステップ完了時の反映方法

- アフィリエイト/ドメイン/AdSense → サイトに実装された時点で翌朝から自動で「完了」判定
- Xアカウント作成 → 次のコマンドでフラグを立てる(またはClaudeに報告):
  ```sh
  echo '{"x_account": true}' > ~/side_hustle_monitor/data/hoshiyomi_manual_done.json
  ```

## 完全自動投稿へのアップグレード(任意)

X APIの無料枠(月500投稿)で毎朝の投稿自体も自動化できる。
必要なもの: Xアカウント作成 → developer.x.com でアプリ作成 → APIキー4点。
キーを渡してもらえれば、毎朝9時のレポート生成時に自動投稿する処理を組み込みます。

## 障害時

- monitor.pyのバックアップ: `monitor.py.bak-20260705-goal`(組み込み前の状態)
- 星詠みのチェックだけ無効化したい場合: main()の checks リストから `check_hoshiyomi` を外す

## 外出先からの投稿(2026-07-07追加)

**https://hoshiyomi365.com/post.html** をスマホで開けば、その日の投稿文+「𝕏で投稿する」ボタンが表示される(noindex・どの端末でもOK)。
生成ロジックはmonitor.pyと同一(14日分の出力一致を検証済み)。Macが手元になくても毎朝の投稿が可能。
スマホのホーム画面に追加しておくと1タップで開ける。
