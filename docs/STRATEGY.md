## ポジショニング

「毎朝3秒の習慣」×「ふたりで試す相性占い」。
無料・登録不要・広告以外の課金なし。ユーザーの行動コストを極限まで下げ、
「毎日変わる」「人に見せたくなる」の2つで再訪とシェアを稼ぐ。

## ファネルと成長ループ

```
認知(SEO / SNSシェア / X運用)
  ↓
体験(今日の運勢・相性占い)← 登録不要なので離脱が少ない
  ↓
シェア(X / LINE、OGP画像で見栄え担保)→ 認知に還流【ループ1】
  ↓
再訪(結果が毎日0時に変わる+入力自動復元)【ループ2】
  ↓
収益(AdSense → アフィリエイト → LINE公式 → 有料鑑定)
```

- **ループ1(シェア)**: 相性占いは「相手に送る理由」が構造的に存在する(=1診断で2人に届く)。シェアURLにはUTM付与済みで、GA4を入れれば流入経路が計測できる。
- **ループ2(習慣化)**: 結果が毎日変わる決定論設計。「今日のふたりの相性運」も毎日変化し、カップル・友人同士の毎日チェックを狙う。
- **ループ3(SEO)**: 「運命数 ○ 意味」「タロット ○○ 逆位置」「相性占い 生年月日 無料」等のロングテールを解説ページで拾い、CTAでアプリへ流す。

## ユーザーにしかできないこと(優先順)

1. **Google Search Console 登録**(無料・5分)
   https://search.google.com/search-console → URLプレフィックスで公開URLを登録 → sitemap.xml を送信。
   SEO流入はここから始まる。最優先。
2. **GA4 導入**(無料・10分)
   analytics.google.com でプロパティ作成 → 測定ID(G-XXXX)を index.html のコメント箇所に貼って解除。
   計測なしでは何も改善できない。
3. **X アカウントでのローンチ投稿**(docs/marketing.md のテンプレ使用)
4. **独自ドメイン**(年1,000円前後)
   AdSense審査の通過率が大きく上がる。お名前.com等で取得し、GitHub PagesのCustom domainに設定。
5. **AdSense申請**(独自ドメイン後)
6. **楽天/Amazonアフィリエイト**(ラッキーアイテムのリンク化)

## KPI(週次で見る数字)

| 指標 | 定義 | 最初の目標 |
|---|---|---|
| UU | 週間ユニークユーザー | 100 |
| 実行率 | 訪問→占い実行 | 60% |
| シェア率 | 実行→シェアクリック | 3% |
| 再訪率 | 7日以内の再訪 | 15% |

※すべてGA4導入後に計測可能。シェア率はUTM流入の逆算でも近似できる。

## 30日ロードマップ

- 週1: Search Console + GA4 + ローンチ投稿。毎朝の運勢投稿を7日続ける(テンプレあり)
- 週2: 独自ドメイン設定、AdSense申請。相性占いの投稿(カップル・推し活文脈)
- 週3: 検索クエリをSearch Consoleで確認し、流入があるテーマの解説ページを増強
- 週4: AdSense掲載開始(結果画面下+解説ページ記事間)、KPI初回レビュー

## 正直な見立て

「爆売れ」を保証する施策は存在しない。この構成が保証するのは、
(1)当たった時に取りこぼさない導線と、(2)外れている場所を特定できる計測。
占いアプリは供給過多のジャンルなので、差別化は「毎日変わる決定論設計」と
「解説コンテンツの質」で作る。数字が動き始めるのはSEOが効き出す4〜8週目以降が普通。
